# Project design context

> Persistent design context for the impeccable skills (arrange, typeset,
> colorize, polish, critique, distill, animate). Fill this in for the project —
> it's read before any design pass. Replace every ⟨placeholder⟩.

## Design Context

### Users
⟨Who uses this, and in what context? What job are they trying to get done?⟩

### Brand Personality
⟨Three words. Voice / tone. Casing, punctuation, emoji policy.⟩

### Aesthetic Direction
⟨The visual tone and references. The one accent, the neutral grounds, the
typeface. Anti-references — what it must NOT look like (e.g. the AI-slop tells:
purple→blue gradients, glowing dark mode, rounded-card grids, glassmorphism,
monospace-as-"technical"). If someone could say "an AI made this," it has failed.⟩

### Design Principles
1. ⟨**Type does the talking** — one family, fluid display, strong weight
   contrast. Typography carries hierarchy before colour does.⟩
2. ⟨**One accent, used with intent** — dominant neutrals; the accent for the one
   most important mark. Other hues are a pinch of spice, never a rainbow.⟩
3. **Surfaces invert, tokens never repeat.** Write `text-foreground` /
   `bg-background` / `border-border` once; a `theme-*` class decides whether
   they resolve light-on-dark or dark-on-light. No per-element colour.
4. **Calm motion.** ease-out for entrances/exits, ease for hover, transform and
   opacity only, no springs or bounce. Motion comes from the shared primitives
   (`Reveal`, `Stagger`, `MaskReveal`) — never per-component easing. Reduced
   motion is honoured by `<MotionConfig reducedMotion="user">`, which drops
   transforms and keeps opacity; **never branch the DOM on `useReducedMotion`**,
   and never add a `prefers-reduced-motion` query that kills opacity too — that
   strands content invisible.
5. **Negative space is the texture.** Flat, hairline-bordered, generous. Remove
   before adding — perfection is when nothing's left to take away.

## Stack conventions (respect, don't reinvent)
- **Tailwind v4** `@theme` for all tokens (`--color-*`, `--text-*` + companions,
  `--radius-*`, `--shadow-*`, `--ease-*`). No arbitrary values in markup.
- **shadcn** token vocabulary (`background/foreground/primary/muted/border/…`),
  overridden per surface via `.theme-*` scopes. Components live in
  `src/components/ui`, added with `bunx shadcn@latest add …`.
- **Next.js** App Router; verify APIs against `node_modules/next/dist/docs`
  before writing — majors have breaking changes.
- Package manager: **bun**.
